# CLAUDE.md

## Task Completion Requirements

- All of `bun fmt`, `bun lint`, and `bun typecheck` must pass before considering tasks completed.
- NEVER run `bun test`. Always use `bun run test` (runs Vitest).
- Build must pass: `bun build:desktop` (builds contracts → web → desktop).

## Project Snapshot

iara-desktop is a workspace manager for builders who use Claude Code. It's a full rewrite of the iara Go CLI in TypeScript/Electron — visual project/task management, Claude Code launcher, browser panel for agents, and dev server integration.

This repository is EARLY WIP (M1 scaffold complete, M2+ pending).

## Core Priorities

1. Correctness and reliability first.
2. Performance — keep Electron responsive.
3. Crossplatform — macOS, Linux, Windows.

## Maintainability

Extract shared logic to separate modules. Don't duplicate across files. Don't be afraid to change existing code. Don't take shortcuts by adding local logic when shared utilities exist.

## Package Roles

- `apps/desktop`: Electron 40 main process. Owns window management, IPC handlers, SQLite+Drizzle database, git operations, and all system-level services. Build: tsdown → CJS in dist-electron/.
- `apps/web`: React 19 + Vite 8 renderer. Owns UI layout, routing (TanStack Router), client state (Zustand), and Tailwind CSS 4 styling. Build: Vite → ESM in dist/.
- `packages/contracts`: Shared TypeScript types for IPC bridge, data models. Schema-only — no runtime logic. Build: tsdown → dual ESM+CJS+DTS.
- `packages/shared`: Shared runtime utilities (git, logging, fs). Explicit subpath exports (`@iara/shared/git`, `@iara/shared/fs`) — no barrel index.

## Architecture Notes

- **Bun = tooling only** (install, dev, scripts). Runtime is Node.js via Electron.
- **SQLite lives in main process only.** All queries go through IPC. Renderer never touches DB directly.
- **TanStack Router `routeTree.gen.ts`** is auto-generated by the Vite plugin during `vite build` or `vite dev`. Never create or edit it manually.
- **Hash history in prod** (`createHashHistory()`) to avoid 404s in packaged builds. Browser history in dev.
- **better-sqlite3 + Electron**: May need `@electron/rebuild` for native module compatibility.
- **syncShellEnvironment**: Required on macOS to import PATH from login shell. No-op on Linux/Windows.
- **Env files**: Live in the project dir (editable in any IDE). Symlinks for globals. Injected via `child_process.spawn` env.

## Dev Workflow

```bash
bun install              # Install all deps
bun dev:desktop          # Launch Electron + Vite dev (hot-reload + hot-restart)
bun build:desktop        # Production build
bun typecheck            # TypeScript check all packages
bun lint                 # oxlint
bun fmt                  # oxfmt (auto-fix)
bun fmt:check            # oxfmt (check only)
bun run test             # vitest all packages
bun run smoke-test       # Electron smoke test
```
